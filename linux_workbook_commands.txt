LINUX PROJECT WORKBOOK
======================
A practical command workbook for Linux development.
Most examples assume Debian/Ubuntu-style Linux. Raspberry Pi-specific commands are marked.

--------------------------------------------------------------------
1) BASIC NAVIGATION
--------------------------------------------------------------------
pwd
# Print current working directory.

ls
# List files in current directory.

ls -l
# Long listing: permissions, owner, size, date.

ls -la
# Long listing including hidden files.

ls -lh
# Human-readable sizes.

cd /path/to/folder
# Change to a specific directory.

cd ..
# Go one level up.

cd ~
# Go to home directory.

cd -
# Go back to previous directory.

clear
# Clear terminal screen.

history
# Show previously used commands.

history | tail -50
# Show last 50 commands.

--------------------------------------------------------------------
2) CREATE, COPY, MOVE, DELETE FILES AND FOLDERS
--------------------------------------------------------------------
touch file.txt
# Create an empty file.

mkdir my_project
# Create one folder.

mkdir -p project/src project/include project/build
# Create nested folders in one command.

cp file.txt backup_file.txt
# Copy a file.

cp -r source_folder backup_folder
# Copy folder recursively.

mv oldname.txt newname.txt
# Rename file.

mv file.txt /path/to/destination/
# Move file.

mv folder1 /path/to/destination/
# Move folder.

rm file.txt
# Delete file.

rm -r folder_name
# Delete folder recursively.

rm -rf folder_name
# Force delete folder and contents. Use carefully.

rmdir empty_folder
# Remove empty folder only.

--------------------------------------------------------------------
3) VIEW AND EDIT FILES
--------------------------------------------------------------------
cat file.txt
# Print whole file.

less file.txt
# View file page by page.

head -20 file.txt
# Show first 20 lines.

tail -20 file.txt
# Show last 20 lines.

tail -f logfile.log
# Follow log in real time.

nano file.txt
# Open file in nano editor.

vim file.txt
# Open file in vim editor.

gedit file.txt
# Open file in GUI text editor if desktop environment is installed.

--------------------------------------------------------------------
4) SEARCHING
--------------------------------------------------------------------
grep "text" file.txt
# Search text inside file.

grep -rn "main" ./src
# Recursive search in folder with line numbers.

find . -name "*.c"
# Find all .c files from current directory.

find . -type f -name "*.service"
# Find all systemd service files.

which python3
# Show location of python3 binary.

whereis gcc
# Show binary/source/man locations.

locate libasound
# Quickly search indexed file database.

--------------------------------------------------------------------
5) FILE PERMISSIONS AND OWNERSHIP
--------------------------------------------------------------------
ls -l
# View file permissions.

chmod +x run.sh
# Make shell script executable.

chmod 644 file.txt
# Owner read/write, group read, others read.

chmod 755 run.sh
# Owner full, others read/execute.

chown user:user file.txt
# Change file owner and group.

sudo chown -R $USER:$USER my_project
# Change owner of folder recursively to current user.

sudo
# Run a command with superuser privileges.

uptime

free -h

mount

7) PROCESS MANAGEMENT
--------------------------------------------------------------------

top

htop

jobs

bg

8) NETWORK COMMANDS
--------------------------------------------------------------------
ip a

ip link
# Show interfaces.

ping 8.8.8.8

11) GCC / C / C++ BUILD COMMANDS
--------------------------------------------------------------------
gcc main.c -o app
# Compile simple C file.

gcc main.c utils.c -o app
# Compile multiple C files.

gcc -Wall -Wextra main.c -o app
# Compile with warnings enabled.

gcc -g main.c -o app
# Compile with debug symbols.

gcc -c main.c -o main.o
# Compile only to object file.

gcc main.o utils.o -o app
# Link object files to executable.

g++ main.cpp -o app
# Compile simple C++ file.

make
# Build project using Makefile.

make clean
# Remove build artifacts if Makefile supports it.

//////compile the with gcc if use lasound
gcc main.c -o app -lasound -lm

13) SHELL SCRIPT BASICS

Example shell script:

python3 main.py
# Simple shell script to move to project directory and run Python.

chmod +x run_python.sh
# Make shell script executable.

./run_python.sh
# Run script from current directory.

bash run_python.sh
# Run script even if not executable.

--------------------------------------------------------------------
14) BETTER BASH FILE TO RUN PYTHON
--------------------------------------------------------------------
Example: run_app.sh
#!/bin/bash
set -e
cd /home/user/my_project
source venv/bin/activate
python3 main.py >> app.log 2>&1

# set -e = stop on first error.
# >> app.log = append output to log.
# 2>&1 = redirect errors also to same log.

Make executable:
chmod +x run_app.sh

Run manually:
./run_app.sh

--------------------------------------------------------------------
15) SYSTEMD SERVICE BASICS
--------------------------------------------------------------------
Check status:
systemctl status myapp.service
# Show service state.

Start service:
sudo systemctl start myapp.service
# Start service now.

Stop service:
sudo systemctl stop myapp.service
# Stop service now.

Restart service:
sudo systemctl restart myapp.service
# Restart service.

Enable service:
sudo systemctl enable myapp.service
# Start automatically at boot.

Disable service:
sudo systemctl disable myapp.service
# Stop auto-start at boot.

Reload service files:
sudo systemctl daemon-reload
# Re-read systemd unit files after editing service.

View logs:
journalctl -u myapp.service -n 100
# Show last 100 lines of service log.

Follow logs live:
journalctl -u myapp.service -f
# Live log output.

List failed services:
systemctl --failed
# Show failed services.

--------------------------------------------------------------------
16) EXAMPLE SYSTEMD SERVICE FOR PYTHON APP
--------------------------------------------------------------------
Create service file:
sudo nano /etc/systemd/system/myapp.service

Example content:
[Unit]
Description=My Python App
After=network.target

[Service]
Type=simple
User=user
WorkingDirectory=/home/user/my_project
ExecStart=/home/user/my_project/run_app.sh
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

Then run:
sudo systemctl daemon-reload
sudo systemctl enable myapp.service
sudo systemctl start myapp.service
systemctl status myapp.service

--------------------------------------------------------------------
21) RASPBERRY PI CONFIG / INTERFACES
--------------------------------------------------------------------
sudo raspi-config
# Open Raspberry Pi configuration tool.

In raspi-config you can enable:
# I2C
# SPI
# SSH
# Audio options (depending on OS)
# Interface settings

Check config file:
sudo nano /boot/firmware/config.txt
# Raspberry Pi OS newer path.

Alternative older path:
sudo nano /boot/config.txt
# Raspberry Pi OS older path.

View current config:
grep -v '^#' /boot/firmware/config.txt | sed '/^$/d'
# Show non-comment active lines only.

Important common config lines:
dtparam=i2c_arm=on
# Enable I2C.

dtparam=spi=on
# Enable SPI.

dtoverlay=hifiberry-dac
# Example I2S DAC overlay.

dtoverlay=i2s-mmap
# Example depending on use case/OS.

dtparam=audio=off
# Disable onboard audio if using external DAC overlay and needed.

Reboot after config change:
sudo reboot
# Apply changes.

3. Ask for systemd service.

35) SAFE HABITS
--------------------------------------------------------------------
# Use pwd before rm -rf.
# Test scripts manually before putting them in systemd.
# Keep backup of config.txt and service files.
# Use journalctl after every service change.
# Use git for your project files.
# Use comments in scripts and this workbook.
# Do not edit /boot or /etc files blindly without backup.

Backup examples:
sudo cp /boot/firmware/config.txt /boot/firmware/config.txt.bak
# Backup Pi config.

sudo cp /etc/systemd/system/myapp.service /etc/systemd/system/myapp.service.bak
# Backup service file.

38) FINAL NOTE
--------------------------------------------------------------------
This workbook is a strong starting point, but some commands depend on:
# Linux distribution
# Raspberry Pi model
# Whether the project is C, C++, Python, or mixed
# Whether audio/I2S is onboard or through an external DAC overlay

For hardware interfaces like I2S, SPI, I2C, and audio, the exact enable/config steps can change by board and OS image.
